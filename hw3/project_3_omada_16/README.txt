Compilation και τρέξιμο:

	Τρέχουμε το σκριπτάκι static_lib για να γίνει compiled η user-level εφαρμογή που καλεί τα system calls
	./static_lib

	Και μετά μπορούμε να χρησιμοποιήσουμε τα system calls
	./get_slob_mem_measurement

- Για να τρέξουν τα 2 system calls βάλαμε στο slab.h τις δηλώσεις των μεταβλητών ως extern long int.

- Το test.c είναι το πρόγραμμα που χρησιμοποιήσαμε κατα τη διάρκεια των δοκιμών μας

Γενικές παρατηρήσεις:

- Στο best-fit implementation για τις σελίδες και τα blocks, η αναλογία Allocated/Free είναι πιο μεγάλη, δηλαδή παρόλο
που δεσμεύεται μεγάλο μέγεθος μνήμης, ο ελεύθερος χώρος είναι ελάχιστος. Το "πριονίδι" που δημιουργείται είναι επίσης ελάχιστο,
αφού το best fit για τις σελίδες φροντίζει ότι θα επιλεχθεί η σελίδα με το καλύτερο δυνατό block. Στις υλοποιήσεις first-fit και
next-fit, η αναλογία είναι αρκετά μικρότερη, που δείχνει τον μεγάλο βαθμό του fragmentation.

-Το boot και η απόκριση του kernel με τον best-fit πυρήνα είναι πιο αργά. Αυτό οφείλεται στο γεγονός ότι στον best-fit διατρέχονται
συνεχώς οι λίστες των pages και των blocks, έτσι επιβαρύνεται περισσότερο το σύστημα λόγω των επιπλέον υπολογισμών. Επιτυγχάνεται
δηλαδή καλύτερη διαχείρηση μνήμης αλλά έρχεται με κόστος στην γενική απόδοση του συστήματος.

Στο παρακάτω σχήμα φαίνονται οι αναλογίες που προαναφέρθηκαν (Οι αριθμοί προέρχονται από διαδοχικές κλήσεις των βοηθητικών system calls).
