Γενική παρατήτηρηση για όλα τα configuration files:
	
Όταν μια διεργασία μπαίνει για πρώτη φορά στην Run Queue, το goodness της
έχει πολύ υψηλές τιμές επειδή ο χρόνος παραμονής της είναι 0. Αργότερα, οι τιμές των goodness ομαλοποιούνται.

Παρατηρήσεις με WaitingRQ:
	
Όταν οι διεργασίες βρίσκονται στην Run Queue για μεγάλο χρονικό διάστημα, τα goodnesses τους τείνουν να 
μειώνονται και να σταθεροποιούνται σε τιμές κοντά στο 1. Αυτό σαν αποτέλεσμα έχει ο αλγόριθμος σε βάθος χρόνου να μετατρέπεται σε 
σε Round Robin.

Παρατηρήσεις χωρίς το WaitingRQ:
	
Το goodness των διεργασιών αλλάζει με πολύ πιο αργό ρυθμό, οπότε είναι περισσότερο πιθανό η τρέχουσα
διεργασία να μονοπωλήσει τη CPU. Όταν πέσει αρκετά το goodness των υπολοίπων διεργασιών, τοτε θα γίνει το context switch σε διεργασία
με χαμηλότερο goodness. Παρατηρούμε δηλαδή ότι χρησιμοποιώντας την μεταβλητή WaitingRQ η εναλλαγή μεταξύ των διεργασιών είναι πιο δίκαιη.

Διαφορές μεταξύ goodness και expected burst:
	
Λαμβάνοντας υπόψιν μόνο το expexted burst, υπάρχει η πιθανότητα διεργασίες με χαμηλά exp. bursts να μονoπωλούν την CPU, κανοντας εναλλαγές
μεταξύ τους, ενώ ήδη περιμένουν άλλες με μεγαλύτερο exp. burst στην Run Queue, οι οποίες παθαίνουν starvation. Το πρόβλημα λύνεται
χρησιμοποιώντας το goodness, αφού πλέον λαμβάνεται υπόψιν η παραμονή των υπολοίπων διεργασιών στην Run Queue.
